## Interpretation

**End-to-end correctness is proven.** The precompile contract `store_code` and
`instantiate` both succeeded, which is only possible if `env.mayo_verify` is in the
allowed-imports set (patch `12-cosmwasm-vm.compatibility.rs`) and the host function
is registered (patches `11`/`13`). All three variants then verified to `valid=true`
through the native host path — the in-image `cosmwasm-crypto-mayo` crate runs the
verification natively instead of as metered Wasm.

**The speedup is real but more modest than the original 7× projection** in
`MAYO_PRECOMPILE_PLAN.md`. Two effects explain the gap:

1. **The measured numbers are *total* tx gas**, not the isolated verifier cost. Each
   tx carries ~150–200k of fixed CosmWasm/SDK overhead (signature check, message
   routing, member load, PK-hash recompute + compare). That overhead is unchanged by
   the precompile, so it dilutes the ratio — most visibly at L1 where the pure
   verifier itself is cheap (MAYO-2 pure ≈ 356k total, of which only a fraction is
   the verify proper).

2. **The host gas schedule is deliberately conservative.** `cosmwasm-crypto-mayo`
   charges fixed VM gas per variant (MAYO-2 = 5M, MAYO-3 = 7M, MAYO-5 = 12M VM gas).
   At L1 that charge roughly offsets the Wasm work it replaces; at L5 the native win
   dominates. These constants are a **tunable knob** — lowering them to track real
   native execution time would widen the L1/L3 margins without affecting correctness.

The clean monotonic trend (1.15× → 1.77× → 2.21× as the parameter set grows) is the
expected signature of a precompile: it wins more as the offloaded computation grows.
MAYO-5 — the NIST Level 5 set that beats Falcon-1024 on signature size — is where it
matters most, and there the precompile more than halves the cost.

## Reproduce

```bash
# 1. Build both flavours (host: WSL, needs cargo + wasm-opt)
cd contracts
cargo build --release --target wasm32-unknown-unknown -p jclaw-credential
wasm-opt -O3 --disable-reference-types \
  target/wasm32-unknown-unknown/release/jclaw_credential.wasm \
  -o ../devnet/jclaw_credential_pure.wasm
cargo build --release --target wasm32-unknown-unknown -p jclaw-credential --features mayo-precompile
wasm-opt -O3 --disable-reference-types \
  target/wasm32-unknown-unknown/release/jclaw_credential.wasm \
  -o ../devnet/jclaw_credential_precompile.wasm

# 2. Boot the MAYO-patched devnet (picks up patches 10-19 automatically)
devnet/scripts/run-devnet.sh

# 3. Deploy both + benchmark
FRESH=1 devnet/scripts/benchmark-mayo-devnet.sh
```
