# BN254 precompile — artefact index

A single-page tour of everything the BN254 work produced, in the order
a reviewer should read them.

## 1. The ask

- [`BN254_PRECOMPILE_CASE.md`](./BN254_PRECOMPILE_CASE.md) —
  the gas-cost analysis that motivates the work: pure-Wasm Groth16
  **371 486 gas measured on `uni-7`** (tx `F6D5774E…5080F4DA`) vs.
  precompile **~187 k gas projected** (EIP-1108 parity; devnet
  benchmark in `../devnet/` pending a run to replace the projection
  with a measurement in `./BN254_BENCHMARK_RESULTS.md`).

## 2. The implementation

- [`../wasmvm-fork/README.md`](../wasmvm-fork/README.md) —
  overview of the fork: crate layout, design choices, safety review.
- [`../wasmvm-fork/cosmwasm-crypto-bn254/`](../wasmvm-fork/cosmwasm-crypto-bn254/) —
  the standalone Rust crate with the three host functions
  (`bn254_add`, `bn254_scalar_mul`, `bn254_pairing_equality`),
  conformance tests, and Criterion benchmarks.
- [`../wasmvm-fork/cosmwasm-std-bn254-ext/`](../wasmvm-fork/cosmwasm-std-bn254-ext/) —
  guest-side extension trait used by contracts until `cosmwasm-std`
  absorbs `Api::bn254_*`.
- [`../wasmvm-fork/patches/`](../wasmvm-fork/patches/) —
  unified-diff patches against `CosmWasm/cosmwasm` v2.2.0 and
  `CosmWasm/wasmvm` v2.2.0 that register the imports, expose the
  guest-side API, and plumb the Go FFI.

## 3. The contract change

- [`../contracts/zk-verifier/README_PRECOMPILE.md`](../contracts/zk-verifier/README_PRECOMPILE.md) —
  how to build the precompile variant of the zk-verifier contract.
- [`../contracts/zk-verifier/src/bn254_backend.rs`](../contracts/zk-verifier/src/bn254_backend.rs) —
  the feature-gated backend: pure-arkworks by default, precompile
  under `--features bn254-precompile`.

## 4. The devnet

- [`../devnet/README.md`](../devnet/README.md) —
  ephemeral single-validator `junod` linked against the patched
  `libwasmvm.a`. Spin up with `./scripts/run-devnet.sh`.
- [`../devnet/Dockerfile`](../devnet/Dockerfile) —
  multi-stage build: patched `cosmwasm-vm`, patched `libwasmvm`,
  `junod` linking against both.
- [`../devnet/scripts/`](../devnet/scripts/) —
  `run`, `stop`, `deploy-zk-verifier`, `benchmark` glue.

## 5. The benchmark

- [`../wavs/bridge/src/benchmark-zk-verifier-devnet.ts`](../wavs/bridge/src/benchmark-zk-verifier-devnet.ts) —
  the harness that runs N `VerifyProof` calls against both contract
  variants and writes the comparison markdown. **Hardened post-Ffern**
  with path canonicalisation, allow-roots for input/output, and a
  size cap on the proof bundle.
- [`./BN254_BENCHMARK_RESULTS.md`](./BN254_BENCHMARK_RESULTS.md) —
  *(generated by the benchmark run; not checked in until the devnet
  has been run at least once.)*
- [`./BN254_BENCHMARK_PROJECTED.md`](./BN254_BENCHMARK_PROJECTED.md) —
  the **interim** projection used by the governance proposal until the
  on-chain measurement lands. Auto-generated by
  `cargo run --release -p cosmwasm-crypto-bn254 --example gas_projection`.
- [`../wasmvm-fork/cosmwasm-crypto-bn254/examples/gas_projection.rs`](../wasmvm-fork/cosmwasm-crypto-bn254/examples/gas_projection.rs) —
  the standalone gas-projection tool that produces `BN254_BENCHMARK_PROJECTED.md`.
- [`../devnet/scripts/transfer-image-from-windows.sh`](../devnet/scripts/transfer-image-from-windows.sh) —
  the helper that lifts a pre-built `junoclaw/junod-bn254:devnet`
  Docker image from a Windows host onto the validator VM and brings the
  devnet up on `localhost:36657`. Use this when the in-VM build path is
  blocked (patch line-ending corruption, MSRV churn, etc.).
- [`../wasmvm-fork/BUILD_AND_TEST.md`](../wasmvm-fork/BUILD_AND_TEST.md) —
  the reproducibility contract. Every step is copy-pasteable.

## 6. The governance track

- [`./JUNO_GOVERNANCE_PROPOSAL_BN254.md`](./JUNO_GOVERNANCE_PROPOSAL_BN254.md) —
  the on-chain signaling proposal, citing prop373 as precedent.
- [`./TECHNICAL_PROPOSAL_BN254_SHAREABLE.md`](./TECHNICAL_PROPOSAL_BN254_SHAREABLE.md) —

- [`./MEDIUM_ARTICLE_AFTER_THE_VOTE.md`](./MEDIUM_ARTICLE_AFTER_THE_VOTE.md) —

  three real security fixes, two self-corrections of premature v4
  shipping, one capability expansion — plus the ZK-sidecar wiring, the
  MCP server, and this BN254 precompile reference implementation. The
  contracts and iterations were written by the project's coding agent
  at the author's direction; an earlier draft misattributed them to
  community member Rattadan and that correction is called out inline.
- [`./WASMVM_BN254_PR_DESCRIPTION.md`](./WASMVM_BN254_PR_DESCRIPTION.md) —
  the upstream PR body, ready to paste into a
  `CosmWasm/cosmwasm` + `CosmWasm/wasmvm` PR.
